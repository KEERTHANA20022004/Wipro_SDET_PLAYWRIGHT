// Promises:
//      A Promise in JavaScript is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value.

// A Promise has three states:

// Pending → initial state, neither success nor failure
// Fulfilled → operation completed successfully
// Rejected → operation failed



// =======================
// setTimeout → runs once after delay
// setInterval → runs repeatedly
// clearInterval → stops interval
// Callback → function passed as argument
// Promise → handles async result (success/failure)
// .then → success handler
// .catch → error handler
// =======================


// =======================
// setTimeout → runs once after delay
// =======================
setTimeout(() => {
  console.log("Runs after 5 sec");
}, 5000);


// =======================
// setInterval → runs repeatedly
// =======================
let id = setInterval(() => {
  console.log("Runs every 2 sec");
}, 2000);

// stop interval
clearInterval(id);


// =======================
// CALLBACK FUNCTION
// =======================
function getData(callback) {
  setTimeout(() => {
    console.log("Data fetched");
    callback();
  }, 1000);
}

getData(() => console.log("Next step"));


// =======================
// PROMISE
// =======================
let myPromise = new Promise((resolve, reject) => {
  let success = true;

  if (success) resolve("Success");
  else reject("Error");
});


// =======================
// .then / .catch
// =======================
myPromise
  .then(res => console.log(res))
  .catch(err => console.log(err));


// =======================
// FETCH DATA (Promise simulation)
// =======================
function fetchData() {
  return new Promise(resolve => {
    setTimeout(() => resolve("Data received"), 2000);
  });
}

fetchData().then(data => console.log(data));


// ========================================================
 
fetch("https://catfact.ninja/facts")      // sends request to API to get cat facts
  .then(res => res.json())               // converts response into JSON format
  .then(data => {                       // handles JSON data after conversion
    console.log(data);                 // prints full API response in console

    const facts = data.data;          // extracts only the facts array from response

    
    // using filter

    const result = facts.filter(item => item.length > 70);              // keeps only facts longer than 70 characters

    return result;                                                      // passes filtered data to next .then()
  }) 
  .then(filteredData => console.log(filteredData))                     // prints filtered facts
  .catch(error => console.error("Error:", error));                     // handles any error from fetch or processing

// 1. Async & Await  
// Used to handle asynchronous operations in a cleaner, synchronous-like way.  
// async makes a function return a promise, and await pauses execution until the promise resolves.

// 2. setTimeout & setInterval  
// setTimeout runs a function once after a specified delay.  
// setInterval repeatedly runs a function at a fixed time interval.

// 3. clearTimeout & clearInterval  
// clearTimeout is used to cancel a scheduled setTimeout before it executes.  
// clearInterval stops a repeating function created by setInterval.

// 4. OOPs in JavaScript  
// JavaScript supports Object-Oriented Programming using objects, classes, and prototypes.  
// It includes concepts like encapsulation, inheritance, and polymorphism for structured coding.

// Encapsulation  
// Binding data and methods together inside a single unit (class).  
// It helps in restricting direct access to data and improves security.

// Inheritance  
// A mechanism where one class acquires properties and methods of another class.  
// It helps in code reusability and reduces duplication.

// Polymorphism  
// The ability of a method to take different forms based on context.  
// Same method name can have different behavior in different classes.

// Abstraction  
// Hiding internal implementation details and showing only essential features.  
// It helps in reducing complexity and improves code maintainability.

// Class  
// A blueprint for creating objects with predefined properties and methods.  
// It acts as a template to define structure and behavior.

// Constructor  
// A special method used to initialize object properties when an object is created.  
// It is automatically called when a new object is instantiated.

Type guarding in type script:

Type Guard = Checking a variable’s type before using it.

Why Type Guarding is Needed?

Example:

let value: string | number;

Here value can be:

string
number

So before using string methods or number operations, we check the type.

npm init playwright@latest

Then I installed the required packages:

npm i express
npm i path
npm i body-parser
npm i dotenv

Inside the Day13 folder, a web application was created under the webapp folder. In that web application, Express.js was configured and required dependencies were installed.

A public folder was created containing:
- index.html
- login.html
- login.css

The login page was created for checking login form functionality.

An .env file was used to store the PORT value:

PORT=23000

The Express server was configured in index.js to serve static files and run the application.

A reusable async function named testLoginForm was created for Playwright automation testing. The function locates username, password, login button, and message elements, fills the login details, clicks the login button, and returns the displayed message text. The function was exported using export default for reusability.

Playwright was used to automate and test the login functionality. Multiple test cases were executed using different username and password combinations.

The test script:
- Opened the login page
- Filled login credentials
- Clicked the login button
- Validated the displayed message using assertions

Different login scenarios such as valid credentials, invalid credentials, and validation errors were tested.

Test cases were executed in:
- Chromium

- WebKit

The testing verified whether the login page and form were working properly across different browsers.
